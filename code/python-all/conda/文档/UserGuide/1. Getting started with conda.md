## 安装

要安装conda，必须首先为您选择正确的安装程序。以下是目前可用的最流行的安装程序:

- `Miniconda`（用这个装）
  - `Miniconda` 是 `Anaconda` 提供的最小安装程序。如果您想自己安装大多数软件包，请使用此安装程序。
- `Anaconda Distribution`
  - `Anaconda Distribution` 是一个功能齐全的安装程序，它附带了一套用于数据科学的软件包，以及 Anaconda Navigator (用于使用 conda 环境的 GUI 应用程序)。

## 简单使用

### 启动 conda

在 windows 开始菜单打开 `Anaconda Powershell Prompt (miniconda3)`

### 创建环境

Conda允许您创建单独的环境，每个环境包含它们自己的文件、包和包依赖项。每个环境的内容彼此之间不交互。

创建新环境的最基本方法是使用以下命令:

```sh
conda create -n <env-name>
```

要在创建环境时添加包，请在环境名称后指定它们:

```sh
conda create -n myenvironment python numpy pandas
```

### 列出环境

看所有环境：

```sh
conda info --envs
```

切换环境：

```sh
conda activate <env-name>
```

切换环境回到默认的环境 `base` ：

```sh
conda activate
```

### 安装包

```sh
# 切换环境然后安装
conda activate myenvironment
conda install matplotlib

# 在 install 命令后添加 name 参数指定环境
conda install --name myenvironment matplotlib
```

### 指定 channel

channel 是存储包的位置(在您自己的计算机上或 Internet 上的其他地方)。默认情况下，conda 在其默认通道中搜索包。

如果你想要的包位于另一个通道，比如 `conda-forge` ，你可以在安装包时手动指定通道:

```sh
conda install conda-forge::numpy
```

您还可以覆盖 `.condarc` 文件中的默认通道。

### 更新 conda

查看 version

```sh
conda --version
```

更新 conda

```sh
conda update conda
```

## other

用 `nancondar powershell` 进行控制

base：代表基础环境。尽量不在基础环境下载库或做其他事。

通常新建一个环境，切换到这个环境，在里面安装 python，再进行操作

命令：

- `condar env list`
  - 查看环境列表
- `condar create --name [name]`
  - 创建环境
- `condar activate [name]`
  - 切换/激活环境
- `condar install python`
  - 安装 python
- `condar list`
  - 查看当前环境安装的库
- `condar install [package name] -c conda-forge`
  - 安装库的 `channel` 切换到 `conda-forge` 这个 `channel`

condar 和 pip 的区别

- 依赖
  - condar 会分析包的依赖，并会下载对应的依赖。
  - pip 只会下载包。下载包下来时有时会用不到，因为没有装对应的依赖
- 版本
  - condar 的库的版本比较旧。所以切换 `channel` 为 `conda-forge`
  - pip 的库的版本都是最新的
