# 验证文件路径

在 api 接口中，如果请求参数有文件路径，然后我们需要拿取文件的时候，简单的做法是直接拿这个路径去访问文件系统

```py
import os
def some_api(file_path):
    if os.path.exists(file_path):
        with open(file_path, 'r') as f:
            data = f.read()
        return data
    else:
        return "File not found"
```

但这样存在安全隐患，比如路径穿越攻击（path traversal attack）。攻击者可能会构造恶意路径，访问系统上不应该被访问的文件，例如服务器上的敏感文件。

```py
import os
def some_api(file_path):
    # 如果 file_path 是 "../etc/passwd"，攻击者就可以访问到系统的 passwd 文件
    if os.path.exists(file_path):
        with open(file_path, 'r') as f:
            data = f.read()
        return data
    else:
        return "File not found"
```

所以在处理文件路径时，我们需要进行验证，确保路径是合法的，并且限制访问范围。

常见的做法是使用一个基目录（base directory），然后将用户提供的路径与这个基目录进行拼接，并验证最终路径是否在基目录下。

```py
import os
def some_api(file_path):
    base_dir = "/allowed/directory"

    # 验证路径是否在 base_dir 下
    if os.path.commonpath([base_dir, full_path]) == base_dir and os.path.exists(full_path):
        with open(full_path, 'r') as f:
            data = f.read()
        return data
    else:
        return "File not found or access denied"
```
