# 大量图片页面优化策略

核心思路：分而治之 + 按需加载

## 1. **图片懒加载（Lazy Loading）**

- **原理**：只加载当前视口内的图片，当用户滚动时再加载即将进入视口的图片，避免一次性发起数百个HTTP请求。
- **实现方式**：
  - 使用现代浏览器的 `loading="lazy"` 属性（简单可靠）。
  - 使用 Intersection Observer API 进行更精细的控制（可自定义加载阈值、占位等）。
- **注意**：配合占位符（如模糊图、骨架屏）提升用户体验。

## 2. **图片格式与压缩优化**

- **格式选择**：
  - 优先使用 **WebP** 或 **AVIF** 格式（同等质量下体积更小），并提供 `<picture>` 降级方案支持旧浏览器。
  - 对于图标类图片，考虑使用 **SVG** 或 **字体图标**。
- **压缩**：
  - 通过工具（如 imagemin、TinyPNG）对图片进行有损或无损压缩。
  - 根据显示尺寸动态生成合适分辨率的图片（响应式图片），避免加载超大原图。

## 3. **网络传输优化**

- **CDN 加速**：将图片托管到 CDN，利用边缘节点加速加载。
- **HTTP/2 或 HTTP/3**：多路复用减少连接开销，提升并发加载效率。
- **缓存策略**：设置强缓存（Cache-Control）和协商缓存，避免重复加载。
- **预加载关键图片**：对于首屏重要的图片，使用 `<link rel="preload">` 提前加载。

## 4. **渲染性能优化**

- **虚拟滚动（Virtual Scrolling）**：如果图片是列表或瀑布流形式，只渲染当前可见区域内的图片元素，移除不可见区域的 DOM 节点，极大减少 DOM 数量。
- **CSS containment**：使用 `contain: content` 或 `content-visibility: auto` 告诉浏览器哪些部分可以独立渲染，减少重绘范围。
- **图片解码控制**：添加 `decoding="async"` 属性让图片解码异步进行，不阻塞主线程。
- **占位符**：先展示低分辨率或纯色背景，避免白屏。

## 5. **内存管理**

- **及时释放 DOM**：虚拟滚动时销毁不可见图表的 DOM，并解除事件绑定，防止内存泄漏。
- **避免大尺寸图片同时存在于内存**：通过 canvas 处理时注意释放资源。

## 6. **并发控制**

- 如果必须一次性加载较多图片（如图库预览），可以手动控制并发请求数量（如用 `p-limit` 库），避免浏览器连接池被占满导致其他资源阻塞。

## 7. **用户体验降级与渐进增强**

- 在弱网环境下，提供“加载更多”按钮让用户主动触发，或使用骨架屏过渡。
- 对于极慢网络，可考虑使用模糊占位图 + 渐进式 JPEG 展示。

## 8. **使用现代浏览器特性**

- `loading="lazy"` 和 `decoding="async"` 是最简单的优化。
- 使用 `Intersection Observer` 比监听 `scroll` 事件性能更好。
- 利用 `requestIdleCallback` 在空闲时加载非关键图片。

## 9. **服务端配合**

- 图片服务支持动态裁剪尺寸（如七牛、又拍云等），根据前端需求返回最合适的尺寸。
- 使用 HTTP/2 Server Push 推送关键图片（需谨慎，避免过度推送）。
