# 登录密码加密

使用 HTTPS 协议，确保数据在传输过程中被加密，防止被窃听和篡改。

> 使用 HTTP 协议时，数据以明文形式传输，容易被攻击者窃听和篡改。而 HTTPS 协议通过使用 SSL/TLS 加密数据，确保数据在传输过程中被保护。
>
> 即使前端使用了加密算法对密码进行加密，如果没有使用 HTTPS，攻击者仍然可以通过网络嗅探工具捕获到加密后的密码，并尝试破解它。因此，使用 HTTPS 是保护用户密码安全的基本措施。

在服务器端，使用安全的哈希算法（如 bcrypt、scrypt 或 Argon2）对密码进行哈希处理，并添加盐值（salt）以增加安全性。这样即使数据库被泄露，攻击者也无法轻易获取用户的原始密码。

***

注册流程：

1. 用户在注册界面输入用户名和密码。
2. 浏览器通过 HTTPS 将用户名和密码发送到服务器。
3. 服务器接收到用户名和密码后，使用安全的哈希算法对密码进行哈希处理，并添加盐值。
4. 服务器将哈希后的密码和盐值存储在数据库中。

登录流程：

1. 用户在登录界面输入用户名和密码。
2. 浏览器通过 HTTPS 将用户名和密码发送到服务器。
3. 服务器接收到用户名和密码后，使用安全的哈希算法对密码进行哈希处理，并添加盐值。
4. 服务器将哈希后的密码与数据库中存储的哈希值进行比较，验证用户身份。
5. 如果验证成功，服务器生成会话或令牌，返回给客户端，用于后续的身份验证。

***

在 chrome 里，查看登录请求的 payload 能看到密码明文，这是正常的。

因为 payload 是浏览器在“加密之前”展示给当前用户自己的内容，所以它能显示用户输入的明文密码。真正加密后的密码是在浏览器将数据通过 HTTPS 发送到服务器时进行加密的，这个过程对用户是透明的。
