# micro service

## 1. 有 docker、微服务的相关经验吗 / 在项目中你是怎么使用 docker、微服务的。有运维的相关经验吗

有相关经验。Docker 这块我主要是用于开发环境统一和项目部署。比如前端项目我会通过多阶段构建把代码打包成静态资源，再配合 Nginx 做成 Docker 镜像，这样测试、预发、生产环境的部署方式比较一致，也方便回滚。

微服务方面，我的理解更多是站在前端视角。前端通常不会直接管理微服务拆分，但会和网关或者 BFF 层协作。我在项目里做过多服务接口联调，处理过鉴权、跨域、接口超时、统一错误处理、页面降级这些问题。如果一个页面依赖多个服务，我会关注数据请求编排和异常兜底，保证用户体验。

运维方面我不是专职做基础设施的，但有实际配合经验，比如查看容器日志、确认环境变量、检查 Nginx 转发、定位发布后接口异常，以及配合 CI/CD 做发布和回滚。
