# 介绍

`ASP.NET` 是微软开发的一个免费、开源的 **Web 框架**，用于构建现代、高性能、可扩展的 Web 应用程序、服务和 API。 它是庞大的 `.NET` 平台 (`.NET Framework` 和 `.NET Core/.NET 5+`) 的一部分，运行在公共语言运行时 之上，允许开发者使用 `C#、F#` 或 `Visual Basic` 等语言进行编码。

## 开发模型

多种开发模型： 支持构建不同类型 Web 应用的不同模式：

- `ASP.NET Web Forms`： 早期的模型，提供类似桌面开发的“拖拽控件”和事件驱动体验，使用 ViewState 管理状态。适合快速开发业务线应用，但有时不够灵活，且 ViewState 可能带来性能开销。
- `ASP.NET MVC`： 基于模型-视图-控制器设计模式，清晰地分离关注点（业务逻辑、UI、用户输入处理）。提供更好的控制力、可测试性和灵活性，是现代 Web 开发的主流选择之一。
- `ASP.NET Web API`： 专门用于构建 RESTful HTTP 服务，轻松创建供 Web、移动、桌面或 IoT 应用消费的 API。通常与 MVC 结合使用或单独使用。
- `ASP.NET Web Pages`： 更轻量级的模型，使用 `Razor` 语法直接在 `HTML` 中嵌入 `C#` 代码。适合小型应用或简单页面。
- `ASP.NET Core`： 这是 ASP.NET 的现代化、跨平台、高性能、开源的重构版本。它整合并优化了 `MVC、Web API、Razor Pages` 等模型，并引入了全新的架构（如中间件管道）。它是当前和未来的发展方向。
- `Blazor`： 一个革命性的框架，允许开发者使用 `C#` 和 `Razor` 语法构建交互式 Web UI，而无需大量 JavaScript。它有两种模式：
  - `Blazor Server`： UI 在服务器上运行，通过 SignalR 实时更新 DOM。
  - `Blazor WebAssembly`： 将 .NET 运行时和应用程序直接下载到浏览器，完全在客户端运行。

## 部署选项

- `Windows Server + IIS`： 传统方式（也适用于 ASP.NET Core）。
- Linux (各种发行版) + `Nginx/Apache`： ASP.NET Core 的常见部署目标。
- macOS： 主要用于开发，也可部署。
- `Docker` 容器： 非常流行的方式，提供一致性和可移植性。
- 云平台： 如 `Microsoft Azure` (App Service, Azure Functions, AKS), `AWS` (Elastic Beanstalk, ECS, EKS), `Google Cloud Platform` (GKE, Cloud Run) 等。
