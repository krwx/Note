## if...else

`condition` 可以是任何能求值为 `true` 或 `false` 的表达式。

```js
if (condition) {
  // 条件为真时执行的代码
} else {
  // 条件为假时执行的代码
}
```

```js
if (condition) {
  // 条件为真时执行的代码
} else if (anotherCondition) {
  // 另一个条件为真时执行的代码
} else {
  // 条件为假时执行的代码
}
```

下面这些值求值为 false（也叫做假值）：

- `false`
- `undefined`
- `null`
- `0`
- `NaN`
- 空字符串（`""`）

所有其他的值——包括所有的对象——会求值为 true。

## switch

```js
switch (expression) {
  case value1:
    // 当 expression 的值与 value1 相等时执行的代码
    break;
  case value2:
    // 当 expression 的值与 value2 相等时执行的代码
    break;
  default:
    // 当 expression 的值与上面所有的 case 都不相等时执行的代码
}
```

如果 `case` 里面没有 `break`，程序会在 `switch` 语句内继续执行（将会执行下一个 `case` 的语句，依此类推）。

```js
switch (expression) {
  case value1:
    // 当 expression 的值与 value1 相等时执行的代码
    // 没有 break，所以会继续执行下面的 case
  case value2:
    // 当 expression 的值与 value2 相等时执行的代码
    break;
  default:
    // 当 expression 的值与上面所有的 case 都不相等时执行的代码
}
```

## try catch

### 基本用法

```javascript
try {
  // 可能会抛出错误的代码
} catch (error) {
  // 处理错误的代码
}
```

### 异步操作说明

在 `try catch` 里面执行异步操作（例如发送 http 请求），但是不使用 `await` 等待，那么 `catch` 里面的代码**不会捕获到异步操作抛出的错误**。

```js
try {
  router.push('/some-page') // 这是一个异步操作
} catch (error) {
  console.error('发生错误:', error);
}
```

使用 `await` 等待异步操作完成后，`catch` 就能捕获到错误了：

```js
try {
  const result = await router.push('/some-page') // 等待导航完成
  if (!result) {
    throw new Error('导航失败')
  }
} catch (error) {
  console.error('发生错误:', error);
}
```

## try finally

### 基本用法

```javascript
try {
  // 可能会抛出错误的代码
} finally {
  // 无论是否发生错误都会执行的代码
}
```

### 异步操作说明

在 try finally 里面执行异步操作（例如发送 http 请求），但是不使用 await 等待，那么 finally 里面的代码**不会等异步操作完成后再执行**。

**1、执行顺序说明**

```javascript
try {
  console.log('1. 开始发送请求');
  fetch('/api/data') // 异步请求
    .then(response => console.log('3. 请求完成'));
} finally {
  console.log('2. finally 执行');
}

// 输出顺序：
// 1. 开始发送请求
// 2. finally 执行
// 3. 请求完成
```

**2、为什么会这样？**

- `fetch()` 是**异步操作**，它会立即返回一个 Promise，而不会阻塞代码执行
- `finally` 块会在 `try` 块中的**同步代码执行完后立即执行**，不会等待异步操作完成
- JavaScript 的事件循环机制决定了异步回调会在后续的微任务/宏任务中执行

**3、如果需要等待请求完成，在 finally 执行代码**

可以使用 `async/await`：

```javascript
async function example() {
  try {
    console.log('1. 开始发送请求');
    await fetch('/api/data');
    console.log('2. 请求完成');
  } finally {
    console.log('3. finally 执行');
  }
}

// 输出顺序：
// 1. 开始发送请求
// 2. 请求完成
// 3. finally 执行
```

**总结**：`finally` 只关心同步执行流程，不关心异步操作的状态。
