## 定义

**MDN 定义：闭包（closure）是由函数以及函数所在的词法环境组合而成的。** 该词法环境包含了这个闭包创建时作用域内的任何局部变量。

**Deepseek 定义：闭包是指 一个函数能够访问其定义时的词法作用域，即使该函数在其定义的作用域之外被调用。**

补充说明：

- 闭包让开发者可以从内部函数访问外部函数的作用域。
- 在 JavaScript 中，闭包会随着函数的创建而同时创建。
  - 当一个函数被创建时，闭包也会被创建。闭包会捕获函数定义时所在的词法环境，并将其与函数一起存储起来。

## 闭包的基础示例

**例子1：**

```js
function init() {
  var name = "Mozilla"; // name 是 init 创建的局部变量
  function displayName() {
    // displayName() 是内部函数，它创建了一个闭包
    console.log(name); // 使用在父函数中声明的变量
  }
  displayName();
}
init();
```

在上面的例子中，**`displayName()` 及其词法环境的组成** 是一个**闭包**，因为它能够访问 `init()` 函数中的变量 `name`。当我们调用 `init()` 函数时，`displayName()` 被创建并且能够访问 `name` 变量。

**例子2：**

```js
function makeFunc() {
  const name = "Mozilla";
  function displayName() {
    console.log(name);
  }
  return displayName;
}

const testFunc = makeFunc();
testFunc();
```

在本例中，`testFunc` 是执行 `makeFunc` 时创建的 `displayName` 函数的实例引用。`displayName` 的实例有一个它的词法环境的引用，而 `name` 变量位于这个词法环境中。因此，当 `testFunc` 被调用时，`name` 变量仍然可用，其值 `Mozilla` 就被传递到 `console.log` 中。

## 闭包作用域链

**1、闭包可以访问其外部函数的作用域链中的变量。**

```js
// 全局作用域
const e = 10;
function sum(a) {
  return function (b) {
    return function (c) {
      // 外部函数作用域
      return function (d) {
        // 局部作用域
        return a + b + c + d + e;
      };
    };
  };
}

console.log(sum(1)(2)(3)(4)); // 20
```

**2、闭包可以捕获块作用域的变量**

```js
function outer() {
  let getY;
  {
    const y = 6;
    getY = () => y;
  }
  console.log(typeof y); // undefined
  console.log(getY()); // 6
}

outer();
```

这里，`getY` 是一个闭包，它捕获了块作用域中的变量 `y`。虽然在 `outer` 函数中无法直接访问 `y`，但通过调用 `getY()`，我们仍然可以访问到 `y` 的值。

**3、闭包可以捕获模块作用域的变量**

```js
// myModule.js
let x = 5;
export const getX = () => x;
export const setX = (val) => {
  x = val;
};
```

这里，模块导出一对 `getter-setter` 函数，它们在模块作用域变量 `x` 上创建了闭包。即便在其他模块中不能直接访问 `x` 的情况下，也能通过函数对 `x` 进行读写。

## 闭包的常见用途

### 1. 用闭包模拟私有方法

利用闭包可以创建私有变量，避免全局污染。

```js
var Counter = (function () {
  var privateCounter = 0;
  function changeBy(val) {
    privateCounter += val;
  }
  return {
    increment: function () {
      changeBy(1);
    },
    decrement: function () {
      changeBy(-1);
    },
    value: function () {
      return privateCounter;
    },
  };
})();

console.log(Counter.value()); /* 0 */
Counter.increment();
Counter.increment();
console.log(Counter.value()); /* 2 */
Counter.decrement();
console.log(Counter.value()); /* 1 */
```

`increment()、decrement()、value()` 这三个公共方法是共享同一个环境的闭包

### 2. 函数工厂

闭包可以用于生成具有特定行为的函数。

```js
function multiplyBy(factor) {
  return function (number) {
    return number * factor;
  };
}

const double = multiplyBy(2);
console.log(double(5)); // 10
```

用途：柯里化函数、防抖 / 节流

### 3. 保留函数变量对象

普通函数的局部变量每次执行都会重新初始化，闭包可以让变量在函数多次调用间保留上一次的执行结果。

```js
// 累加器：每次调用都在上一次结果上累加
function createAdder(init) {
  let total = init; // 保留累加的状态
  return (num) => {
    total += num;
    return total;
  };
}
 
const add5 = createAdder(0);
console.log(add5(2)); // 2
console.log(add5(3)); // 5（保留了上一次的total=2）
console.log(add5(5)); // 10
 
const add10 = createAdder(10);
console.log(add10(5)); // 15（和add5的total相互独立）
```

## 注意事项

使用的危害可能有：

1. 内存泄漏：闭包长期持有外层函数的大数据或 DOM 引用，导致无法被垃圾回收。
2. 性能开销：访问外部变量需沿作用域链查找，创建闭包本身也更耗内存和 CPU。
3. 循环变量共享陷阱：在循环中用 var 创建闭包时，所有闭包共享最终值（let 可避免）。

### 1. 内存泄漏

原因：闭包保留了对 定义函数时的外部作用域 的引用，导致外部函数的变量无法被 GC 回收，最终引发内存泄漏。

以下是几种典型的由闭包引起内存泄漏的情况。

#### 1.1. 闭包持有大型数据或 DOM 元素

当一个闭包持有了一个大型对象（例如大数组、Buffer）或 DOM 元素的引用，并且这个闭包被长期保留（例如作为全局回调、事件监听器），那么这些数据将无法被回收，即使它们已经不再需要。

```javascript
function createHeavyClosure() {
  const hugeData = new Array(1000000).fill('*'); // 占用大量内存
  const domElement = document.getElementById('leak'); // DOM 引用

  return function() {
    // 虽然闭包内部可能只使用了 domElement，但 hugeData 也被闭包环境捕获
    console.log(domElement.id);
  };
}

const leakyFunc = createHeavyClosure(); // 闭包一直存在
// 即使 hugeData 从未被使用，它依然被保留在闭包的作用域链中
```

**原因**：闭包会保留其**整个词法环境**，其中包含了外部函数中的所有变量（不仅仅是显式使用的变量）。如果外部函数中有大量未使用但体积巨大的数据，这些数据也会被闭包锁定。

**解决方案**：

- 只在闭包中引用真正需要的变量。

    ```js
    function createLogger(user) {
      const userName = user.name;

      return function () {
        console.log(userName);
      };
    }

    const user = {
      name: 'Tom',
      cache: new Array(10000).fill('data'),
    };

    const logger = createLogger(user);
    logger();
    // 闭包只保留 userName，不再持有整个 user 大对象
    ```

- 及时释放引用：在闭包使用完毕后，将闭包变量设为 `null` 以解除引用。

    ```js
    let closure = (function() {
      let bigData = new Array(10000);
      return function() { console.log(bigData.length); };
    })();
    // 不再需要时
    closure = null;  // bigData 可被回收
    ```

- 使用**模块模式**或**立即执行函数（IIFE）**，并确保在退出时清理。

    ```js
    // 使用立即执行函数
    const handler = (function () {
      const config = { url: '/api/user', timeout: 3000 };

      return function () {
        console.log(config.url);
      };
    })();

    handler();
    // config 不会暴露到外部全局作用域，减少无意义的长期引用
    ```

#### 1.2 闭包在定时器或全局回调中未被清理

```javascript
function startTimer() {
  let secret = 'sensitive data';
  setInterval(function() {
    console.log('Tick...');
    // 虽然没有使用 secret，但闭包捕获了它
  }, 1000);
}
startTimer();
```

**问题**：`setInterval` 的回调函数形成了一个闭包，它捕获了 `secret` 变量。即使 `secret` 从未被使用，它也会一直留在内存中，直到定时器被清除。如果定时器无限运行，`secret` 就永远无法释放。

**解决方案**：

- 只捕获必要的变量。
- 在定时器不再需要时调用 `clearInterval`，并确保解除回调对外部变量的引用。

#### 1.3 事件监听器没有解绑

先看一个事件监听器没有解绑导致内存泄漏的例子：

```js
function bindButton(button) {
  function handleClick() {
    console.log(button.textContent);
  }

  button.addEventListener('click', handleClick);
}

const button = document.getElementById('submit');
bindButton(button);

// 后续如果 button 从页面中移除，但事件监听没有解绑，
// button 和 handleClick 之间的引用关系就可能让相关对象继续存活
```

为什么这可能导致内存泄漏：`button` 通过事件系统持有 `handleClick`，而 `handleClick` 这个闭包又引用了外部变量 `button`。如果 DOM 节点已经不再使用，但事件监听器一直没有移除，这条引用链就可能让节点和闭包都无法及时被垃圾回收。

解决思路：在不再需要这个 DOM 元素时，主动移除事件监听，并断开闭包中对 DOM 的引用，避免形成长期存活的引用链。

解决方案示例：

```js
function bindButton(button) {
  function handleClick() {
    console.log(button.textContent);
  }

  button.addEventListener('click', handleClick);

  return function destroy() {
    if (button) {
      button.removeEventListener('click', handleClick);
      button = null;
    }
  };
}

const button = document.getElementById('submit');
const destroy = bindButton(button);

// 在组件卸载或页面销毁时清理
destroy();
```

#### 1.4 缓存回收问题

先看一个普通 `Map` 带来的缓存回收问题：

```js
const cache = new Map();

function getUserProfile(user) {
  if (!cache.has(user)) {
    cache.set(user, {
      fullName: `${user.firstName} ${user.lastName}`,
    });
  }

  return cache.get(user);
}

let user = { firstName: 'Tom', lastName: 'Lee' };
console.log(getUserProfile(user));

user = null;
// 即使外部不再使用 user，Map 里的 key 仍然强引用这个对象
// 对应缓存不会自动释放，除非手动 cache.delete(user)
```

上面的问题在于：`Map` 会对 key 保持**强引用**。如果你忘了手动清理缓存，即使外部代码已经不再使用这个对象，这条缓存仍然可能继续留在内存里。

如果你的需求是“**给对象建立一份额外缓存，但对象销毁后缓存也能自动释放**”，那么 `WeakMap` / `WeakSet` 就很适合。

它们的关键特点是：**对对象的引用是弱引用**。这意味着当对象本身没有其他强引用时，垃圾回收器可以回收这个对象，同时与之关联的 `WeakMap` 项或 `WeakSet` 成员也会一起失效，不需要手动删除。

例子：

```js
const cache = new WeakMap();

function getUserProfile(user) {
  if (!cache.has(user)) {
    cache.set(user, {
      fullName: `${user.firstName} ${user.lastName}`,
    });
  }

  return cache.get(user);
}

let user = { firstName: 'Tom', lastName: 'Lee' };
console.log(getUserProfile(user));

user = null;
// user 没有其他强引用后，对应的 WeakMap 项也可被回收
```

### 2. 闭包中变量是引用传递

闭包保存的是外部变量的引用，而非变量的值。内部函数修改外部变量会直接改变原变量。

```js
function createObj() {
  const obj = { name: '张三' };
  return {
    get: () => obj.name,
    set: (newName) => { obj.name = newName; }
  };
}

const fn = createObj();
fn.set('李四');
console.log(fn.get()); // 李四 → 闭包引用了obj，修改的是原对象
```

### 3. 循环中创建闭包

ES5 中 `var` 声明的变量没有块级作用域，循环中创建闭包会导致所有闭包共享同一个变量，最终结果都是变量的最后一个值

```js
// 坑：var 声明，所有定时器共享同一个 i，先执行完 for 循环，然后再执行 setTimeout 里的回调，最终输出 10 个 10
for (var i = 0; i < 10; i++) {
  setTimeout(() => {
    console.log(i); // 输出10个10
  }, 100);
}
```
  
ES6 中用 `let` 声明（块级作用域）可直接解决，或用立即执行函数（`IIFE`）为每个闭包创建独立作用域。

```js
// 解决1：ES6 let 声明（推荐），块级作用域，每个循环都有独立的 i
for (let i = 0; i < 10; i++) {
  setTimeout(() => {
    console.log(i); // 输出 0-9
  }, 100);
}
 
// 解决2：ES5 立即执行函数，为每个闭包创建独立作用域
for (var i = 0; i < 10; i++) {
  (function(j) {
    setTimeout(() => {
      console.log(j); // 输出 0-9
    }, 100);
  })(i); // 立即执行，传递当前 i 的值
}
```
