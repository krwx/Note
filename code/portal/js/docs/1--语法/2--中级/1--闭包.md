## 定义

**MDN 定义：闭包（closure）是由函数以及函数所在的词法环境组合而成的。** 该词法环境包含了这个闭包创建时作用域内的任何局部变量。

**Deepseek 定义：闭包是指 一个函数能够访问其定义时的词法作用域，即使该函数在其定义的作用域之外被调用。**

补充说明：

- 闭包让开发者可以从内部函数访问外部函数的作用域。
- 在 JavaScript 中，闭包会随着函数的创建而同时创建。
  - 当一个函数被创建时，闭包也会被创建。闭包会捕获函数定义时所在的词法环境，并将其与函数一起存储起来。

## 闭包的基础示例

**例子1：**

```js
function init() {
  var name = "Mozilla"; // name 是 init 创建的局部变量
  function displayName() {
    // displayName() 是内部函数，它创建了一个闭包
    console.log(name); // 使用在父函数中声明的变量
  }
  displayName();
}
init();
```

在上面的例子中，**`displayName()` 及其词法环境的组成** 是一个**闭包**，因为它能够访问 `init()` 函数中的变量 `name`。当我们调用 `init()` 函数时，`displayName()` 被创建并且能够访问 `name` 变量。

**例子2：**

```js
function makeFunc() {
  const name = "Mozilla";
  function displayName() {
    console.log(name);
  }
  return displayName;
}

const testFunc = makeFunc();
testFunc();
```

在本例中，`testFunc` 是执行 `makeFunc` 时创建的 `displayName` 函数的实例引用。`displayName` 的实例有一个它的词法环境的引用，而 `name` 变量位于这个词法环境中。因此，当 `testFunc` 被调用时，`name` 变量仍然可用，其值 `Mozilla` 就被传递到 `console.log` 中。

## 闭包作用域链

**1、闭包可以访问其外部函数的作用域链中的变量。**

```js
// 全局作用域
const e = 10;
function sum(a) {
  return function (b) {
    return function (c) {
      // 外部函数作用域
      return function (d) {
        // 局部作用域
        return a + b + c + d + e;
      };
    };
  };
}

console.log(sum(1)(2)(3)(4)); // 20
```

**2、闭包可以捕获块作用域的变量**

```js
function outer() {
  let getY;
  {
    const y = 6;
    getY = () => y;
  }
  console.log(typeof y); // undefined
  console.log(getY()); // 6
}

outer();
```

这里，`getY` 是一个闭包，它捕获了块作用域中的变量 `y`。虽然在 `outer` 函数中无法直接访问 `y`，但通过调用 `getY()`，我们仍然可以访问到 `y` 的值。

**3、闭包可以捕获模块作用域的变量**

```js
// myModule.js
let x = 5;
export const getX = () => x;
export const setX = (val) => {
  x = val;
};
```

这里，模块导出一对 `getter-setter` 函数，它们在模块作用域变量 `x` 上创建了闭包。即便在其他模块中不能直接访问 `x` 的情况下，也能通过函数对 `x` 进行读写。

## 闭包的常见用途

### 1. 用闭包模拟私有方法

利用闭包可以创建私有变量，避免全局污染。

```js
var Counter = (function () {
  var privateCounter = 0;
  function changeBy(val) {
    privateCounter += val;
  }
  return {
    increment: function () {
      changeBy(1);
    },
    decrement: function () {
      changeBy(-1);
    },
    value: function () {
      return privateCounter;
    },
  };
})();

console.log(Counter.value()); /* 0 */
Counter.increment();
Counter.increment();
console.log(Counter.value()); /* 2 */
Counter.decrement();
console.log(Counter.value()); /* 1 */
```

`increment()、decrement()、value()` 这三个公共方法是共享同一个环境的闭包

### 2. 函数工厂

闭包可以用于生成具有特定行为的函数。

```js
function multiplyBy(factor) {
  return function (number) {
    return number * factor;
  };
}

const double = multiplyBy(2);
console.log(double(5)); // 10
```

用途：柯里化函数、防抖 / 节流

### 3. 保留函数变量对象

普通函数的局部变量每次执行都会重新初始化，闭包可以让变量在函数多次调用间保留上一次的执行结果。

```js
// 累加器：每次调用都在上一次结果上累加
function createAdder(init) {
  let total = init; // 保留累加的状态
  return (num) => {
    total += num;
    return total;
  };
}
 
const add5 = createAdder(0);
console.log(add5(2)); // 2
console.log(add5(3)); // 5（保留了上一次的total=2）
console.log(add5(5)); // 10
 
const add10 = createAdder(10);
console.log(add10(5)); // 15（和add5的total相互独立）
```

## 注意事项

### 1. 内存泄漏

原因：闭包保留了对外部作用域的引用，导致外部函数的变量无法被 GC 回收，最终引发内存泄漏。

以下是几种典型的由闭包引起内存泄漏的情况。

#### 1.1. 闭包持有大型数据或 DOM 元素

当一个闭包持有了一个大型对象（例如大数组、Buffer）或 DOM 元素的引用，并且这个闭包被长期保留（例如作为全局回调、事件监听器），那么这些数据将无法被回收，即使它们已经不再需要。

```javascript
function createHeavyClosure() {
  const hugeData = new Array(1000000).fill('*'); // 占用大量内存
  const domElement = document.getElementById('leak'); // DOM 引用

  return function() {
    // 虽然闭包内部可能只使用了 domElement，但 hugeData 也被闭包环境捕获
    console.log(domElement.id);
  };
}

const leakyFunc = createHeavyClosure(); // 闭包一直存在
// 即使 hugeData 从未被使用，它依然被保留在闭包的作用域链中
```

**原因**：闭包会保留其**整个词法环境**，其中包含了外部函数中的所有变量（不仅仅是显式使用的变量）。如果外部函数中有大量未使用但体积巨大的数据，这些数据也会被闭包锁定。

**解决方案**：

- 只在闭包中引用真正需要的变量。
- 在闭包使用完毕后，将闭包变量设为 `null` 以解除引用。
- 使用**模块模式**或**立即执行函数（IIFE）**，并确保在退出时清理。

#### 1.2 闭包在定时器或全局回调中未被清理

```javascript
function startTimer() {
  let secret = 'sensitive data';
  setInterval(function() {
    console.log('Tick...');
    // 虽然没有使用 secret，但闭包捕获了它
  }, 1000);
}
startTimer();
```

**问题**：`setInterval` 的回调函数形成了一个闭包，它捕获了 `secret` 变量。即使 `secret` 从未被使用，它也会一直留在内存中，直到定时器被清除。如果定时器无限运行，`secret` 就永远无法释放。

**解决方案**：

- 只捕获必要的变量。
- 在定时器不再需要时调用 `clearInterval`，并确保解除回调对外部变量的引用。

### 2. 闭包中变量是引用传递

闭包保存的是外部变量的引用，而非变量的值。内部函数修改外部变量会直接改变原变量。

```js
function createObj() {
  const obj = { name: '张三' };
  return {
    get: () => obj.name,
    set: (newName) => { obj.name = newName; }
  };
}

const fn = createObj();
fn.set('李四');
console.log(fn.get()); // 李四 → 闭包引用了obj，修改的是原对象
```

### 3. 循环中创建闭包

ES5 中 `var` 声明的变量没有块级作用域，循环中创建闭包会导致所有闭包共享同一个变量，最终结果都是变量的最后一个值

```js
// 坑：var 声明，所有定时器共享同一个 i，先执行完 for 循环，然后再执行 setTimeout 里的回调，最终输出 10 个 10
for (var i = 0; i < 10; i++) {
  setTimeout(() => {
    console.log(i); // 输出10个10
  }, 100);
}
```
  
ES6 中用 `let` 声明（块级作用域）可直接解决，或用立即执行函数（`IIFE`）为每个闭包创建独立作用域。

```js
// 解决1：ES6 let 声明（推荐），块级作用域，每个循环都有独立的 i
for (let i = 0; i < 10; i++) {
  setTimeout(() => {
    console.log(i); // 输出 0-9
  }, 100);
}
 
// 解决2：ES5 立即执行函数，为每个闭包创建独立作用域
for (var i = 0; i < 10; i++) {
  (function(j) {
    setTimeout(() => {
      console.log(j); // 输出 0-9
    }, 100);
  })(i); // 立即执行，传递当前 i 的值
}
```
