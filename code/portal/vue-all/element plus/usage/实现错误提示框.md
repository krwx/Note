下面是一个使用 Vue3 和 Element Plus 实现的全局请求错误提示框组件方案，支持在其他组件中调用其 `open` 方法设置标题和内容：

## 1. 创建错误提示框组件

**路径: `src/components/ErrorModal.vue`**

```vue
<template>
  <el-dialog
    v-model="visible"
    :title="title"
    width="30%"
    destroy-on-close
    :show-close="false"
  >
    <div class="error-content">
      <el-icon color="#F56C6C" :size="20">
        <Warning />
      </el-icon>
      <span>{{ content }}</span>
    </div>
    
    <template #footer>
      <el-button type="primary" @click="visible = false">确定</el-button>
    </template>
  </el-dialog>
</template>

<script setup lang="ts">
import { ref } from 'vue'
import { Warning } from '@element-plus/icons-vue'

const visible = ref(false)
const title = ref('错误提示')
const content = ref('')

// 暴露给外部调用的方法
const open = (errorTitle: string, errorContent: string) => {
  title.value = errorTitle
  content.value = errorContent
  visible.value = true
}

// 暴露 open 方法
defineExpose({ open })
</script>

<style scoped>
.error-content {
  display: flex;
  align-items: center;
  gap: 10px;
}
</style>
```

## 2. 在组件中使用

```vue
<template>
    <ErrorModal ref="errorModal" />
</template>

<script setup>
import ErrorModal from './ErrorModal.vue';

const errorModalRef = ref<InstanceType<typeof ErrorModal> | null>(null);

const fetchData = async () => {
  try {
    await $axios.get('/api/data')
  } catch (error) {
    errorModalRef.value?.open('请求错误', error.message)
  }
}
</script>
```
