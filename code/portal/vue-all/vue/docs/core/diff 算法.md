Vue 的 diff 算法是其虚拟 DOM 核心的一部分，用于高效地将更新后的虚拟 DOM 树与旧的虚拟 DOM 树进行比较，从而以最小的代价将变化应用到真实 DOM 上。下面我将从背景、核心思想、具体实现、优化策略以及与 React diff 的对比等方面详细介绍。

---

## 1. 为什么需要 diff 算法？

在 Vue 中，数据变化会触发重新渲染，生成新的虚拟 DOM 树。如果直接替换整个真实 DOM，代价太高。因此需要一种高效的比较算法，找出两棵树之间的差异，然后只更新需要改变的部分。这就是 diff 算法的目标。

## 2. 核心思想：同层比较 + 双端比较

Vue 的 diff 算法遵循以下基本原则：

- **只进行同层比较**：不会跨层级比较节点。如果节点跨层级移动，Vue 会直接销毁旧节点并重新创建新节点（因为实际场景中跨层级移动很少，这样做可以简化算法，保证效率）。
- **时间复杂度 O(n)**：通过策略优化，将传统树 diff 的 O(n³) 复杂度降低到 O(n)。
- **基于策略的优化**：
  - 如果节点类型不同，直接替换。
  - 如果节点类型相同，则复用 DOM 元素，并更新属性（patchVNode）。
  - 对于同一层级的子节点列表，使用 **双端比较** 算法。

## 3. 双端比较算法详解

Vue 2.x 的 diff 算法在比较同一层级的子节点列表时，采用了 **双端比较**（也称为“头尾指针法”）。它通过四个指针分别指向新旧两个子节点列表的头和尾，然后进行四种比较尝试：

- **旧头 vs 新头**：如果相同，直接复用节点，两个头指针后移。
- **旧尾 vs 新尾**：如果相同，直接复用节点，两个尾指针前移。
- **旧头 vs 新尾**：如果相同，说明旧头节点移动到了尾部，将对应 DOM 元素移动到尾部，旧头指针后移，新尾指针前移。
- **旧尾 vs 新头**：如果相同，说明旧尾节点移动到了头部，将对应 DOM 元素移动到头部，旧尾指针前移，新头指针后移。

如果以上四种情况都不匹配，则使用新头节点的 `key` 去旧节点列表中查找相同 `key` 的节点：

- 如果找到且节点类型相同，则复用并移动到头部，然后将旧节点列表的原位置置为 `undefined` 避免重复使用。
- 如果没找到或类型不同，则创建新节点插入到头部。

重复上述过程，直到任一列表的头指针超过尾指针。最后，如果旧节点列表还有剩余，则批量删除；如果新节点列表还有剩余，则批量添加。

**图示流程（简化）：**

```txt
旧列表：[A, B, C, D]
新列表：[B, A, D, C]

步骤：
1. 旧头A vs 新头B → 不同
   旧尾D vs 新尾C → 不同
   旧头A vs 新尾C → 不同
   旧尾D vs 新头B → 不同 → 用key查找：B在旧列表索引1，将B移动到真实 DOM 头部，旧列表变为[A, undefined, C, D]（实际是标记），新头指针后移。真实 DOM 变化：[B A C D]
2. 旧头A vs 新头A → 相同（A未移动，但需跳过已处理的B），复用A，旧头指向C，新头指向D。
3. 旧头C vs 新头D → 不同
   旧尾D vs 新尾C → 不同
   旧头C vs 新尾C → 相同 → 将C移动到尾部，旧头指向D，新尾指向D。真实 DOM 变化：[B A D C]
4. 旧头D vs 新头D → 相同 → 复用D，结束。
```

## 4. Key 的作用与重要性

在 Vue 中，`key` 是给每个 vnode 的唯一标识。它在 diff 算法中至关重要：

- **准确识别节点**：当节点类型相同但 `key` 不同时，Vue 会认为它们是不同的节点，从而重新创建，而不是复用。
- **避免就地复用带来的副作用**：没有 `key` 或使用索引作为 `key` 时，Vue 会采用“就地复用”策略，可能导致状态错乱（例如输入框内容保留在错误的位置）。
- **提高 diff 效率**：`key` 使得 Vue 能在双端比较失败时，快速通过 `key` 找到可复用的节点，避免不必要的节点创建和删除。

**最佳实践**：使用唯一的、稳定的 `key`（如数据的 id），避免使用数组索引。

## 5. Vue 3 中的优化

Vue 3 对 diff 算法做了进一步优化，主要体现在：

- **静态提升**：将静态节点提升到渲染函数外部，避免每次渲染都重新创建。
- **补丁标志（Patch Flags）**：在编译阶段标记动态属性，diff 时只对比带标记的部分，粒度更细。
- **Block Tree**：通过结构化指令（如 v-if、v-for）构建动态节点块，跳过静态子树比较。
- 对于子节点列表的 diff，Vue 3 采用了基于 **最长递增子序列** 的算法来优化移动操作，比双端比较在某些场景下更高效。

尽管如此，核心思想仍然是同层比较和 key 的作用。

## 6. Vue diff 与 React diff 的简要对比

| 特性 | Vue (2.x) | React |
| ------ | ----------- | ------- |
| 比较策略 | 双端比较（头尾指针） | 从左到右逐一比较 |
| 列表 diff 复杂度 | 较低，通过 key 快速定位 | 需要将子节点转为 map，再遍历 |
| 移动检测 | 在比较过程中直接处理移动 | 需要根据 key 映射和索引判断移动 |
| 适用场景 | 适合大量节点移动的场景 | 对节点移动较敏感，可能触发更多重建 |

Vue 的双端比较在移动节点时通常比 React 更高效，因为它能在一次遍历中同时处理头尾的移动。但 React 的 Fiber 架构允许中断渲染，两者各有优势。
