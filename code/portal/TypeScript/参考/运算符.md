# 运算符

## satisfies

`satisfies` 是 TypeScript 4.9 引入的一个运算符，用于**检查一个表达式的类型是否符合某个类型**，同时**保留该表达式最精确的推断类型**。

**1. 基本语法**

```typescript
const value = expression satisfies Type;
```

**2. 为什么需要 `satisfies`？**

有时我们只想检查一个对象是否满足某个接口或类型约束。

但是使用普通的类型注解（`: Type`）会将变量的类型固定为注解的类型，丢失原始表达式更具体的信息。

```typescript
type Colors = "red" | "green" | "blue";
type RGB = [red: number, green: number, blue: number];
const palette: Record<Colors, string | RGB> = {
    red: [255, 0, 0],
    green: "#00ff00",
    bleu: [0, 0, 255]
};

// 这里我们想使用 string.toUpperCase()，但是这里会报错：property 'toUpperCase' does not exist on type 'string | RGB'.
const greenNormalized = palette.green.toUpperCase();
```

所以使用 `satisfies` ，它允许你**检查**类型兼容性，**但保留字面量或表达式的原始类型**：

```typescript
type Colors = "red" | "green" | "blue";
type RGB = [red: number, green: number, blue: number];
const palette = {
    red: [255, 0, 0],
    green: "#00ff00",
    blue: [0, 0, 255]
} satisfies Record<Colors, string | RGB>;

// 这里可以使用 toUpperCase()
const greenNormalized = palette.green.toUpperCase();

// 但是也还是不能访问对象没有定义的属性
const redFirst = palette.yellow; // Error: Property 'yellow' does not exist on type '{ red: number[]; green: string; blue: number[]; }'
```
