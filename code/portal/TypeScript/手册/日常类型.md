## 类型别名（ type alias ）

类型别名：任何类型的名称。类型别名的语法是：

```js
type Point = {
  x: number;
  y: number;
};
 
// Exactly the same as the earlier example
function printCoord(pt: Point) {
  console.log("The coordinate's x value is " + pt.x);
  console.log("The coordinate's y value is " + pt.y);
}
 
printCoord({ x: 100, y: 100 });
```

可以使用类型别名来为任何类型命名

## 接口

接口声明是命名对象类型的另一种方式，与类型别名类似：

```js
interface Point {
  x: number;
  y: number;
}
 
function printCoord(pt: Point) {
  console.log("The coordinate's x value is " + pt.x);
  console.log("The coordinate's y value is " + pt.y);
}
 
printCoord({ x: 100, y: 100 });
```

### 类型别名和接口的区别

1、**声明合并 (Declaration Merging)**

`interface` 支持声明合并

```typescript
interface User {
  name: string;
}
interface User {
  age: number;
}
// 最终 User 包含 name 和 age
const user: User = { name: 'John', age: 25 };
```

`type` 不支持声明合并，会报错

```ts
type User = { name: string };
type User = { age: number }; // 错误：重复标识符
```

2、**扩展方式**

`interface` 使用 `extends` 扩展

```typescript
interface Animal {
  name: string;
}
interface Dog extends Animal {
  breed: string;
}
```

`type` 使用 `&`（交叉类型）扩展

```ts
type Animal = {
  name: string;
};

type Dog = Animal & {
  breed: string;
};
```

2.1、**type 和 interface 可以互相扩展**

```ts
// type 扩展 interface
interface User {
  name: string;
}
type Admin = User & {
  permissions: string[];
};

// interface 扩展 type
type Person = {
  age: number;
};
interface Employee extends Person {
  department: string;
}
```

3、**灵活性**

`type` 更灵活，可以定义多种类型，而 `interface` 主要用于对象类型

```typescript
// type 更灵活，可以定义：
// 1. 联合类型
type ID = string | number;

// 2. 元组类型
type Point = [number, number];

// 3. 字面量类型
type Direction = 'left' | 'right' | 'up' | 'down';

// 4. 条件类型
type IsString<T> = T extends string ? true : false;

// 5. 映射类型
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

// interface 主要用于对象类型
```

**总结建议**

- **优先使用 interface**：当定义对象类型、需要声明合并、或为库提供公共 API 时
- **使用 type**：当需要联合类型、元组、映射类型或复杂类型操作时
- **保持一致**：项目中保持统一的风格
