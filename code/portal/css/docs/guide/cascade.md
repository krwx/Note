## 介绍

判断哪个规则应用于元素的过程叫做层叠（Cascade）。当多个规则适用于同一元素时，浏览器会根据以下步骤来决定哪个规则生效：

1. **是否有 `!important`？**
    - 有：比较来源（作者 > 用户 > 用户代理）。`!important` 的作者样式胜出。
    - 无：进入下一步。
2. **比较来源**：
    - 普通作者样式 > 普通用户样式 > 用户代理样式。
3. **比较特异性**：
    - 内联样式 > ID > 类/属性/伪类 > 元素/伪元素。
4. **比较顺序**：
    - 后写的覆盖先写的。

## 1. 样式来源与重要性

CSS 声明可以有不同的来源：

1. **用户代理样式**：浏览器为元素设置的默认样式（例如 `<a>` 标签默认是蓝色带下划线）。
2. **用户样式**：用户通过浏览器设置自定义的样式（通常很少见，除非是辅助功能需求）。
3. **作者样式**：开发者编写的 CSS（这是我们在开发中最常打交道的部分）。

层叠顺序：

- **作者样式中的 `!important`**（最高优先级）
- **用户样式中的 `!important`**（次高，通常用于无障碍需求，如强制大字体）
- **用户代理样式中的 `!important`**（极少见）
- 普通（非 `!important`）的作者样式
- 普通的用户样式
- 普通的用户代理样式

## 2. 优先级

发生冲突的是规则，需要比较的是规则所在的选择器的优先级。

一个选择器的优先级可以说是由三个不同的值（或分量）相加，可以认为是百（ID）十（类）个（元素）——三位数的三个位数：

- ID：选择器中包含 **ID 选择器**则百位得一分。
- 类：选择器中包含**类选择器、属性选择器或者伪类**则十位得一分。
- 元素：选择器中包含**元素、伪元素选择器**则个位得一分。

> 通用选择器（*）、组合符（+、>、~、' '）和 `:where()` 不会影响优先级。  
>
> `:not()` 和 `:is()` 伪类本身对优先级没有影响，但它们的参数则会带来影响。参数中，对优先级算法有贡献的参数的优先级的最大值将作为该伪类选择器的优先级。

### 内联样式

内联样式，即 style 属性内的样式声明，优先于所有普通的样式，无论其优先级如何。

### `!important`

作者样式中的 `!important` 优先级（从高到低）：

1. 内联样式 + `!important`
2. 选择器规则 + `!important`
3. 普通内联样式（无 `!important`）
4. 普通选择器规则（无 `!important`）

> 少用 `!important`

## 3. 资源顺序

如果两个规则的**来源相同且特异性完全相同**，那么后声明的规则会覆盖先声明的规则。

资源顺序仅在规则的优先级相同时才体现出来
