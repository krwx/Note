## DATE_SUB

`DATE_SUB` 是一个用于**日期减法运算**的函数，它可以从一个日期/时间值中减去指定的时间间隔（如天、月、小时等），返回一个新的日期/时间。

**1、标准语法**

```sql
DATE_SUB(date, INTERVAL expr unit)
```

- **`date`**：起始日期/时间表达式（如 `'2026-05-31'`、`NOW()`）。
- **`INTERVAL`**：固定关键字，表示时间间隔。
- **`expr`**：数值，表示要减去的数量（可为负数，相当于加）。
- **`unit`**：时间单位，如 `DAY`、`MONTH`、`YEAR`、`HOUR`、`MINUTE`、`SECOND`、`WEEK`、`QUARTER` 等。

**返回值**：与 `date` 参数相同类型的日期/时间值。

**2、基本示例**

```sql
-- 减去 10 天
SELECT DATE_SUB('2026-05-31', INTERVAL 10 DAY);     -- 结果: 2026-05-21

-- 减去 3 个月
SELECT DATE_SUB('2026-05-31', INTERVAL 3 MONTH);    -- 结果: 2026-02-28 (自动处理月末)

-- 减去 1 年
SELECT DATE_SUB('2026-05-31', INTERVAL 1 YEAR);     -- 结果: 2025-05-31

-- 减去 2 小时 30 分钟 (复合单位需用更精确的写法)
SELECT DATE_SUB('2026-05-31 08:00:00', INTERVAL '2:30' HOUR_MINUTE);
-- 结果: 2026-05-31 05:30:00

-- 使用负数实现“加”
SELECT DATE_SUB('2026-05-31', INTERVAL -5 DAY);     -- 结果: 2026-06-05
```

**3、常见 `unit` 取值**

| unit 值 | 说明 | 示例写法 |
| --- | --- | --- |
| `MICROSECOND` | 微秒 | `INTERVAL 1000 MICROSECOND` |
| `SECOND` | 秒 | `INTERVAL 30 SECOND` |
| `MINUTE` | 分钟 | `INTERVAL 10 MINUTE` |
| `HOUR` | 小时 | `INTERVAL 2 HOUR` |
| `DAY` | 天 | `INTERVAL 7 DAY` |
| `WEEK` | 周 | `INTERVAL 2 WEEK` |
| `MONTH` | 月 | `INTERVAL 3 MONTH` |
| `QUARTER` | 季度（3 个月） | `INTERVAL 1 QUARTER` |
| `YEAR` | 年 | `INTERVAL 1 YEAR` |
| 复合单位 | 如 `HOUR_MINUTE` | `INTERVAL '2:30' HOUR_MINUTE` |

**4、与其他数据库的对比**

- **MySQL / MariaDB**：完全支持 `DATE_SUB` 函数。
- **PostgreSQL**：没有 `DATE_SUB`，使用减法运算符 `-` 配合 `INTERVAL`：

  ```sql
  SELECT '2026-05-31'::DATE - INTERVAL '10 days';
  SELECT '2026-05-31'::DATE - 10;   -- 直接减天数
  ```

- **SQL Server**：使用 `DATEADD` 函数（单位在前，负值表示减）：

  ```sql
  SELECT DATEADD(day, -10, '2026-05-31');
  ```

- **Oracle**：直接用 `-` 减数字（天数）或 `ADD_MONTHS` 处理月：

  ```sql
  SELECT DATE '2026-05-31' - 10 FROM DUAL;
  SELECT ADD_MONTHS(DATE '2026-05-31', -3) FROM DUAL;
  ```

- **SQLite**：使用 `DATE` 函数配合字符串：

  ```sql
  SELECT DATE('2026-05-31', '-10 days');
  ```

**5、实用场景**

- **查询过去 7 天的订单**：  

  ```sql
  SELECT * FROM orders WHERE order_date > DATE_SUB(CURDATE(), INTERVAL 7 DAY);
  ```

- **计算用户注册后 30 天的提醒日**：

  ```sql
  SELECT DATE_SUB(reg_date, INTERVAL -30 DAY) AS remind_date FROM users;
  ```

- **处理有效期倒计时**：结合 `DATEDIFF` 使用。

**6、注意事项**

- 当月、年减法时，自动处理月末边界（如 `2026-05-31` 减 1 个月得 `2026-04-30`）。
- `INTERVAL` 后面的 `expr` 可以是列名或表达式。
- 若 `date` 参数非法（如 `'2026-02-30'`），结果可能为 `NULL` 或报错（取决于数据库严格模式）。
- 对于时间部分，减法会正确处理跨日/跨月。
