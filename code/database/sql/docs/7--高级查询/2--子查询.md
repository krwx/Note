## 介绍

子查询是指嵌套在另一个 SQL 语句（如 `SELECT`、`INSERT`、`UPDATE`、`DELETE`）中的查询。它可以将一个查询的结果作为另一个查询的条件、数据源或计算字段。子查询也称为**内层查询**，包含它的查询称为**外层查询**。

## 子查询的基本特征

- 必须用括号 `()` 包裹。
- 通常位于比较运算符（`=`、`>`、`IN` 等）的右侧，或作为 `FROM` 子句中的临时表。
- 可以返回单个值、单行、单列或多行多列（取决于使用场景）。

## 典型使用场景及示例

假设已有表：

- `employees` (id, name, salary, department_id)
- `departments` (id, name)

### 1. WHERE 子句中使用子查询

1、查询工资高于平均工资的员工。

```sql
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);
```

2、查询属于销售部或市场部的员工。

```sql
SELECT name, department_id
FROM employees
WHERE department_id IN (
    SELECT id FROM departments WHERE name IN ('Sales', 'Marketing')
);
```

3、查询工资高于任意一个市场部员工的员工（即比市场部最差的高就行）。

```sql
SELECT name, salary
FROM employees
WHERE salary > ANY (
    SELECT salary FROM employees WHERE department_id = 2   -- 假设市场部 ID=2
);
```

### 2. SELECT 子句中使用标量子查询

为每个员工显示其所在部门的平均工资（相关子查询）。

```sql
SELECT name, salary,
       (SELECT AVG(salary) FROM employees e2 
        WHERE e2.department_id = e1.department_id) AS dept_avg
FROM employees e1;
```

### 3. FROM 子句中使用表子查询（派生表）

查询部门平均工资高于 5000 的部门。

```sql
SELECT dept_id, avg_salary
FROM (
    SELECT department_id AS dept_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department_id
) AS dept_avg
WHERE avg_salary > 5000;
```

### 4. EXISTS / NOT EXISTS（常用于相关子查询）

查询至少有一个员工的部门。

```sql
SELECT id, name
FROM departments d
WHERE EXISTS (
    SELECT 1 FROM employees e WHERE e.department_id = d.id
);
```

### 5. 在 DML 语句中使用子查询

- **INSERT**：将查询结果插入另一张表。

  ```sql
  INSERT INTO high_salary_employees (name, salary)
  SELECT name, salary FROM employees WHERE salary > 8000;
  ```

- **UPDATE**：根据其他表的值更新。

  ```sql
  UPDATE employees
  SET salary = salary * 1.1
  WHERE department_id IN (SELECT id FROM departments WHERE name = 'Sales');
  ```

- **DELETE**：删除符合条件的行。

  ```sql
  DELETE FROM employees
  WHERE department_id IN (SELECT id FROM departments WHERE name = 'Temp');
  ```

## 子查询的注意事项

1. **性能问题**：相关子查询会对外层每一行执行一次内层查询，数据量大时效率较低。可考虑用 `JOIN` 或窗口函数改写。
2. **返回多行问题**：若子查询返回多行，但外层使用 `=` 等要求单值的运算符，会报错。应改用 `IN`、`ANY`、`ALL` 等。
3. **NULL 值处理**：`NOT IN` 子查询如果内层结果包含 `NULL`，整个条件会返回 `NULL`（即无行）。建议使用 `NOT EXISTS` 替代。

   ```sql
   -- 危险示例（若子查询有 NULL，则永远无结果）
   SELECT * FROM employees WHERE department_id NOT IN (SELECT id FROM departments);
   -- 安全写法
   SELECT * FROM employees e WHERE NOT EXISTS (SELECT 1 FROM departments d WHERE d.id = e.department_id);
   ```

4. **可读性**：多层嵌套会使语句难以理解，应尽量控制层数，或拆分成 `WITH` 子句（CTE）。

## 子查询 vs 联接（JOIN）

| 场景 | 子查询适用 | JOIN 适用 |
| ------ | ----------- | ----------- |
| 需要聚合结果作为条件 | 标量子查询（如 `WHERE salary > (SELECT AVG(...))`） | 可用窗口函数或派生表 JOIN |
| 检查存在性 | `EXISTS` / `NOT EXISTS` | 可用 `LEFT JOIN ... WHERE ... IS NULL` |
| 从其他表更新/删除 | 子查询更直观 | 某些数据库支持 `UPDATE ... FROM` 或 `JOIN` |
| 多表取不同粒度的数据 | 相关子查询可以避免笛卡尔积 | 需要小心处理聚合后再 JOIN |

通常情况下，能用 `JOIN` 实现且性能更好的尽量使用 `JOIN`，但子查询在某些表达逻辑上更清晰。
