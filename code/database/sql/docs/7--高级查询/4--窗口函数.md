SQL 窗口函数（Window Function）是一种强大的分析工具，它可以在**不改变数据行数**的前提下，对每一行数据执行跨行计算（如排名、累计和、移动平均等）。与普通聚合函数（GROUP BY）不同，窗口函数会保留所有原始记录，并在每一行上返回一个计算结果。

## 1. 窗口函数的核心语法

**1、语法**

```sql
<窗口函数名>(列) OVER (
    PARTITION BY 分组列   -- 可选，定义窗口分区
    ORDER BY 排序列       -- 可选，定义窗口内排序
    ROWS/RANGE 范围定义    -- 可选，限制窗口的行范围
)
```

**2、参数说明**

- **OVER** 关键字：标识这是一个窗口函数。
- **PARTITION BY**：将数据分成多个独立窗口，相当于将数据按指定列分组（类似 GROUP BY，但不合并行）。
- **ORDER BY**：指定窗口内行的顺序，按指定列排序。
  - 如果指定了 `ORDER BY`，但没有指定窗口帧，默认使用 `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`。
  - 如果没有 `ORDER BY`，窗口函数会在整个分区上计算。
- **窗口帧（Frame）**：更进一步限定窗口内参与计算的行子集
  - ROWS：基于物理行数的范围（比如 `ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING`）。
  - RANGE：基于值范围的范围（比如 `RANGE BETWEEN INTERVAL 2 PRECEDING AND CURRENT ROW`。这样理解：`ORDER BY` 会对值排序，然后会选择在区间 `[value - 2, value]` 的行作为窗口）。

**3、窗口帧使用例子**

- `ROWS BETWEEN 1 PRECEDING AND CURRENT ROW`：当前行和前1行。
- `RANGE BETWEEN INTERVAL '7' DAY PRECEDING AND CURRENT ROW`：当前行和前7天内的行（适用于日期类型）。
- `ROWS BETWEEN n PRECEDING AND m FOLLOWING`：当前行向前 n 行到向后 m 行（基于物理行号）。
- `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`：从分区第一行到当前行（累积窗口）。
- `ROWS BETWEEN CURRENT ROW AND UNBOUNDED FOLLOWING`：从当前行到分区最后一行。
- `ROWS BETWEEN n PRECEDING AND CURRENT ROW`：当前行及之前 n 行（常用的滚动窗口）。

### 1.1 ROWS 和 RANGE 的区别

- `ROWS` 是基于行数的范围，直接指定前后多少行。
- `RANGE` 是基于值的范围，指定前后多少值（适用于数值或日期类型）。
  - 窗口包含所有与当前行排序列的值在一定差值范围内的行（通常与 `ORDER BY` 列配合）。
  - 如果 ORDER BY 列有重复值，RANGE 会将这些重复值视为同一行。

**关键区别示例**:

假设有表 `score`：

| id | score |
|----|-------|
| 1  | 80    |
| 2  | 80    |
| 3  | 90    |
| 4  | 100   |

执行以下查询（窗口帧为 `BETWEEN 1 PRECEDING AND CURRENT ROW`）：

```sql
SELECT 
    id, score,
    SUM(score) OVER (ORDER BY score
                     ROWS BETWEEN 1 PRECEDING AND CURRENT ROW) AS rows_sum,
    SUM(score) OVER (ORDER BY score
                     RANGE BETWEEN 1 PRECEDING AND CURRENT ROW) AS range_sum
FROM score;
```

**结果**：

| id | score | rows_sum  | range_sum |
|----|-------|---------- |-----------|
| 1  | 80    | 80        | 160       |
| 2  | 80    | 80+80=160 | 160       |
| 3  | 90    | 90+80=170 | 90        |
| 4  | 100   | 100+90=190| 100       |

**解释**：

- **`ROWS`**：严格取**前1行物理行 + 当前行**。
  - id=1：前1行不存在 → 只取自己（80）。
  - id=2：前1行是 id=1（80），加上自己（80）→ 160。
  - id=3：前1行是 id=2（80），加上自己（90）→ 170。
  - id=4：前1行是 id=3（90），加上自己（100）→ 190。

- **`RANGE`**：取所有 **`score` 值在 `[当前行score - 1, 当前行score]`** 范围内的行。
  - id=1（score=80）：值在 [79,80] 的行 → id=1和2（都是80）→ 160。
  - id=2（score=80）：同样范围 → id=1,2 → 160。
  - id=3（score=90）：范围 [89,90] → 只有 id=3（90）→ 90。
  - id=4（score=100）：范围 [99,100] → 只有 id=4（100）→ 100。

## 2. 常用窗口函数分类

### 2.1 聚合窗口函数

将 SUM、AVG、COUNT、MAX、MIN 等聚合函数用作窗口函数。结果会保留每行，并给出当前窗口的累计或滚动值。

```sql
SELECT 
    employee_id,
    salary,
    SUM(salary) OVER (PARTITION BY department_id) AS dept_total_salary, -- 按部门分组，计算部门总工资
    AVG(salary) OVER (PARTITION BY department_id) AS dept_avg_salary -- 按部门分组，计算部门平均工资
FROM employees;
```

### 2.2 排名函数

- `ROW_NUMBER()`：按顺序编号，相同排名的行获得不同序号（1,2,3,…）。
- `RANK()`：相同排名占用相同序号，但下一个排名会跳跃（1,1,3,…）。
- `DENSE_RANK()`：相同排名相同序号，不跳跃（1,1,2,…）。
- `NTILE(n)`：将数据分成 n 个桶，返回桶编号。

```sql
SELECT 
    student_name,
    score,
    ROW_NUMBER() OVER (ORDER BY score DESC) AS row_num,
    RANK()       OVER (ORDER BY score DESC) AS rank,
    DENSE_RANK() OVER (ORDER BY score DESC) AS dense_rank
FROM exam_scores;
```

### 2.3 偏移函数（取值函数）

- `LAG(列, offset, default)`：返回当前行**之前**第 offset 行的值。
- `LEAD(列, offset, default)`：返回当前行**之后**第 offset 行的值。
- `FIRST_VALUE(列)`：窗口内第一行的值。
- `LAST_VALUE(列)`：窗口内最后一行的值（需注意默认窗口帧范围）。

```sql
SELECT 
    date,
    sales,
    LAG(sales, 1, 0) OVER (ORDER BY date) AS prev_day_sales, -- 前一天的销售额
    LEAD(sales, 1, 0) OVER (ORDER BY date) AS next_day_sales -- 后一天的销售额
FROM daily_sales;
```

## 3. 与普通聚合函数的区别

|特性|普通聚合 + GROUP BY|窗口函数|
|---|---|---|
|输出行数|每个分组输出一行|保留原始每一行|
|能否访问原始细节|不能，需重新关联回原表|直接在同一行上同时显示原始值和汇总值|
|是否需要关联自身表|计算累计/排名时常需自连接|无需自连接，直接计算|
|典型用途|汇总报告（总销售额等）|排名、同比、环比、移动平均|

## 4. 完整示例

4.1、对于销售表 `sales`（字段：region, salesperson, amount），计算每个销售员的销售额占所在区域总销售额的百分比，并按区域排名。

```sql
SELECT 
    region,
    salesperson,
    amount,
    SUM(amount) OVER (PARTITION BY region) AS region_total,
    ROUND(100.0 * amount / SUM(amount) OVER (PARTITION BY region), 2) AS pct_of_region,
    RANK() OVER (PARTITION BY region ORDER BY amount DESC) AS region_rank
FROM sales
ORDER BY region, region_rank;
```

输出示例：

| region | salesperson | amount | region_total | pct_of_region | region_rank |
|--------|-------------|--------|--------------|---------------|-------------|
| East   | Alice       | 1000   | 1800         | 55.56         | 1           |
| East   | Bob         | 800    | 1800         | 44.44         | 2           |
| West   | Carol       | 1200   | 2000         | 60.00         | 1           |
| West   | Dave        | 800    | 2000         | 40.00         | 2           |

---

4.2、使用 `ROWS` 指定窗口大小

```sql
SELECT 
    date,
    sales,
    SUM(sales) OVER (ORDER BY date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS rolling_7_day_sales -- 计算当前行及前6行的销售总和
FROM daily_sales;
```

---

4.3、使用 `RANGE` 指定窗口大小

```sql
SELECT 
    date,
    sales,
    SUM(sales) OVER (ORDER BY date RANGE BETWEEN INTERVAL '6' DAY PRECEDING AND CURRENT ROW) AS rolling_7_day_sales -- 计算当前行及前6天的销售总和
FROM daily_sales;
```

## 5. 注意事项

1. **窗口函数不能直接用在 WHERE 或 HAVING 子句中**。如果需要过滤窗口结果，可使用子查询或 CTE。
2. **ORDER BY 对聚合窗口函数的影响**：  
   如果没有指定窗口帧，`ORDER BY` 会默认使用 `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`（从分区第一行到当前行）。  
   如果没有 `ORDER BY`，聚合窗口函数会在整个分区上计算。
3. **性能**：窗口函数通常比自连接或子查询更高效，但涉及大量分区和排序时仍需注意优化。
